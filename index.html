Firebase 프로젝트 만들기 (약 5분) 1. https://console.firebase.google.com 접속 → Google 계정 로그인 2. **[프로젝트 추가]** → 이름 예: `class-pulse` → Google 애널리틱스는 **사용 안 함** → 만들기 3. 왼쪽 메뉴 **빌드 > Realtime Database** → **[데이터베이스 만들기]** - 위치: **asia-southeast1 (싱가포르)** 선택 - 보안 규칙: **테스트 모드로 시작** 선택 → 사용 설정 4. Realtime Database 화면의 **규칙(Rules)** 탭 → 아래로 교체 후 **게시**: ```json { "rules": { ".read": true, ".write": true } } ``` > ⚠ 이 규칙은 주소를 아는 누구나 읽고 쓸 수 있는 **수업 데모용 개방 설정**입니다. 학기 운영 시에는 만료일 있는 규칙이나 인증 규칙으로 강화하는 것을 권장합니다. 민감한 개인정보(실명·학번)는 저장하지 마세요 — 앱은 익명 닉네임만 사용합니다. ## 2단계. 설정값 복사해서 붙여넣기 (약 2분) 1. Firebase 콘솔 좌측 상단 **⚙ 프로젝트 설정** → 아래 **내 앱** 섹션 → **웹 앱 추가( `` 아이콘)** → 별명 입력(예: class-pulse) → 등록 2. 표시되는 `firebaseConfig` 값을 확인합니다: ```js const firebaseConfig = { apiKey: "AIzaSyCWAEvBAeM7lWNEv949r_Rh6RZ6J9VUBHs", authDomain: "class-pulse-69b1c.firebaseapp.com", databaseURL: "https://class-pulse-69b1c-default-rtdb.asia-southeast1.firebasedatabase.app", projectId: "class-pulse-69b1c", storageBucket: "class-pulse-69b1c.firebasestorage.app", messagingSenderId: "230030559910", appId: "1:230030559910:web:9ceb6bdf18b7f3df040267" }; ``` > `databaseURL`이 안 보이면 Realtime Database 화면 상단의 주소(`https://…firebasedatabase.app`)를 복사해 직접 추가하세요. **databaseURL은 필수입니다.** 3. `class-pulse-live.html` 파일을 텍스트 편집기(VS Code 등)로 열고, 상단의 ★ 표시된 `FIREBASE_CONFIG` 블록에 각 값을 붙여넣고 저장합니다. 4. (선택) 같은 위치의 `ROOM` 값을 강좌별로 바꾸면 데이터가 분리됩니다. 예: `"edu101-2026-2"` ## 3단계. 배포 (약 3분) **방법 A — Vercel (기존에 사용 중이므로 권장)** 1. 파일명을 `index.html`로 바꿔 새 폴더에 넣기 2. GitHub에 새 저장소(예: `jyjung-hub/class-pulse-live`)를 만들어 업로드 3. Vercel에서 해당 저장소 Import → Framework: **Other** → Deploy 4. 발급된 주소(예: `https://class-pulse-live.vercel.app`)가 수업용 링크입니다 **방법 B — GitHub Pages** 1. 저장소에 `index.html` 업로드 → Settings > Pages > Branch: main → Save 2. `https://jyjung-hub.github.io/저장소명/` 이 수업용 링크입니다 **로컬 테스트**: 배포 전 `index.html`을 브라우저로 그냥 열어도 동작합니다(Firebase 설정만 되어 있으면). ## 4단계. 수업에서 사용하기 | 순서 | 교수자 | 학생 | |---|---|---| | 준비 | 배포 주소 접속 → 🔒 → PIN(기본 **1234**) → 교수자 모드 | — | | 시작 | [▶ 퀴즈 세션 시작] → 대기실의 **QR·참여 코드**를 프로젝터에 표시 | QR 스캔 → [▶ 퀴즈] 탭 → 코드 입력 → [입장하기] | | 확인 | **입장한 학생 명단이 즉시(1초 내) 갱신**됨 → 전원 확인 후 [퀴즈 시작] | "입장 완료 ✓" 표시 | | 진행 | 문항별 응답 분포 실시간 확인 → [마감·정답 공개] → [다음 문항] | 제한시간 내 보기 탭 → 정답·해설 확인 | | 상시 | 홈/신호등 탭에서 이해도 게이지·익명 질문 확인 | 신호등(🟢🟡🔴)·익명 질문 | | 종료 | Exit 탭에서 결과 확인, [세션 데이터 초기화]로 다음 수업 준비 | Exit Ticket 3문항 제출 | **첫 사용 시 반드시**: 교수자 홈 > 설정에서 **PIN 변경**(기본 1234), 참여 코드 변경. ## 아티팩트 버전과의 차이 | | 아티팩트 버전 (claude.ai) | LIVE 버전 (독립 배포) | |---|---|---| | 실시간 동기화 | 계정 간 공유 불가 확인됨 | **Firebase 실시간 (1초 내)** | | 학생 접속 | Claude 로그인 필요 | **로그인·설치 불필요** | | AI 문항 생성·군집화 | 가능 | 미포함 (API 키 노출 방지) | | 용도 | 설계·시연·문항 제작 | **실제 수업 진행** | > AI 문항 생성은 수업 전에 아티팩트 버전(또는 이 대화)에서 만들고, LIVE 버전의 [+ 문항 직접 추가]로 옮기는 워크플로를 권장합니다. 필요하면 Exit Ticket 서술 응답을 이 대화에 붙여넣어 군집화 분석을 요청하셔도 됩니다. ## 학기 운영 (LIVE 2.0) - **주차 선택**: 교수자 홈 상단의 1~15주 버튼으로 현재 주차를 지정합니다. 퀴즈·신호등·질문·Exit·분석·플로우 진행 데이터가 **주차별로 완전히 분리** 저장되며, 학생 화면도 자동으로 해당 주차를 따라갑니다. 8주(중간)·15주(기말)는 🎓 표시와 평가 주간 안내가 나타납니다. - **성적 반영 퀴즈**: 대기실을 열 때 "성적에 반영 / 반영 안 함"을 선택합니다. 성적 반영 퀴즈는 대기실·학생 화면에 🎯 표시가 붙고, 학생에게 표시이름을 **학번+이름**으로 설정하라는 안내가 나옵니다(학생 홈 [표시이름 설정]). - **기록 조건**: 퀴즈를 마지막 문항까지 진행해 "결과 보기"에 도달해야 해당 주차 기록으로 저장됩니다. 중간에 [세션 종료]하면 기록되지 않습니다. - **Excel 출력** (분석 탭): - [📥 N주차 활동 Excel]: 해당 주차의 퀴즈기록·신호등·익명질문·ExitTicket 4개 시트 - [🎯 성적반영 퀴즈 종합 Excel]: 전체 15주에서 성적 반영 퀴즈만 모아 **주차별요약 / 개인별 / 문항별** 3개 시트로 출력 — 학기말 성적 산출용 - **개인정보 주의**: 성적 반영 시 학번·이름이 DB에 저장되므로, 데모용 개방 규칙 대신 Firebase 보안 규칙을 강화한 뒤 운영하시기를 권장합니다. ## 문제 해결 - **"최초 설정이 필요합니다" 화면** → FIREBASE_CONFIG가 아직 기본값입니다. 2단계 확인. - **헤더에 🔴 연결 끊김** → databaseURL 오타 또는 네트워크 문제. Realtime Database 주소와 일치하는지 확인. - **입장해도 명단에 안 뜸** → 규칙(Rules)이 1단계 4번처럼 `.read/.write: true`로 게시되었는지 확인. - **여러 강좌 운영** → ROOM 값을 강좌별로 다르게 한 파일을 각각 배포하거나, 학기마다 [세션 데이터 초기화] 사용.
